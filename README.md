# ①課題名
マーケティングプロンプトメーカーver3
〜Geminiによるプロンプト自動翻訳機能付き〜

## ②課題内容（どんな作品か）
マーケティングの業務に活用するためのプロンプトを作成するためのWebアプリ

以下、挙動
①新規登録画面でメールアドレスとパスワードを入力 ※ユーザー毎にユニークなIDを付与
②フォームに自社サービスの情報（概要、ブランド、ターゲット）を入力 ※ここではまだプロンプト品質は粗い
③入力情報をGeminiへ飛ばし、プロンプトを精度高く翻訳 ⭐️新規追加
④Geminiからプロンプトを受け取り、Webアプリ側に表示 ⭐️新規追加
⑤FirebaseのFireStoreへプロンプトを保存 ※ユニークID単位（=ユーザー単位）でプロンプトが保存される

## ⑤工夫した点・こだわった点
- 卒業プロダクトに実装予定の「入力内容を生成AIに翻訳させ出力する」のプロトタイプ第一弾
- GeminiAPIへデータ送信し、受信結果をFireStoreに保存した点
- 前回実装した機能（Firebaseへのデータ保存）をそのまま活用できた点

## ⑥難しかった点・次回トライしたいこと（又は機能）
- GeminiのAPIへの接続が中々うまくいかず、APIの色々なversionを何度も行き来きしエラーを潰していった（404、429エラーなど色々出ました）
- 「await fetch」と「awaitのみ」の違いが完全には理解できてないこと
- Geminiへのデータ送信には文法のルールがあるらしく、ここは生成AIにかなり頼ってしまったこと
